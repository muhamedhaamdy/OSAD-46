Introduction to
Unit Testing

A beginner-friendly guide for developers writing JavaScript for the first time with a testing mindset.

Why What goes wrong without tests?
Real scenario — no tests
You build a discount calculator. It works. Six months later, a teammate changes the tax logic. Suddenly discounts break — but nobody notices until customers complain. You spend 2 days hunting a 3-line bug.
Silent regressions
Old code breaks when new code is added. Without tests, you only find out in production.
Fear of refactoring
Teams avoid improving messy code because they can't tell if they broke something.
Manual testing loops
Every change requires clicking through the app again. Slow, error-prone, and boring.
No shared truth
Different developers have different assumptions about what the code is supposed to do.
Types The testing pyramid
E2E TestsSlow · Few · Expensive
Integration TestsMedium speed
Unit Tests ← we are hereFast · Many · Cheap
Unit tests
Test a single function or module in complete isolation. No database, no network, no UI. Just inputs and outputs. Fast to write, instant to run.

Example: does calculateDiscount(100, 20) return 80?

Integration tests
Test how two or more modules work together. May involve a real database or API call. Slower than unit tests, but catches "the parts don't fit" bugs.

Example: does the checkout module correctly talk to the inventory module to reduce stock?

End-to-end (E2E) tests
Simulate a real user clicking through your app in a real browser. Tests the whole stack — frontend, backend, database. Very realistic but also slow and brittle.

Example: open Chrome, add item to cart, checkout, check confirmation email was sent.

Use cases When do you write unit tests?
Pure functions (math, formatting) Validation logic Business rules (pricing, discounts) String manipulation Data transformations Edge cases (null, 0, empty array) Error handling Utility helpers
✓
Confidence to refactor — change the internals of a function, run tests, know immediately if you broke anything.
⚡
Instant feedback — unit tests run in milliseconds. You don't wait for a browser or a server to tell you something is wrong.
📖
Living documentation — a test suite tells future developers exactly what a function is supposed to do in every scenario.
🐛
Catch bugs early — find mistakes during development, not after deployment when the cost is much higher.
Code Starting code — JavaScript

The function we want to test:

// math.js 
function calculateDiscount(price, discountPercent) {
  if (price < 0) throw new Error('Price cannot be negative');
  if (discountPercent < 0 || discountPercent > 100) {
    throw new Error('Discount must be between 0 and 100');
  }
  return price * (1 - discountPercent / 100);
}

module.exports = { calculateDiscount };

The unit tests (using Jest):

// math.test.js 
const { calculateDiscount } = require('./math');

// A "test suite" groups related tests
describe('calculateDiscount', () => {

  // Happy path
  test('applies 20% discount to $100', () => {
    const result = calculateDiscount(100, 20);
    expect(result).toBe(80);
  });

  // Edge case: 0% discount
  test('returns original price when discount is 0', () => {
    expect(calculateDiscount(50, 0)).toBe(50);
  });

  // Edge case: 100% discount (free)
  test('returns 0 when discount is 100%', () => {
    expect(calculateDiscount(200, 100)).toBe(0);
  });

  // Error cases
  test('throws on negative price', () => {
    expect(() => calculateDiscount(-10, 20))
      .toThrow('Price cannot be negative');
  });

  test('throws when discount exceeds 100', () => {
    expect(() => calculateDiscount(100, 150))
      .toThrow('Discount must be between 0 and 100');
  });

});

Set up in your project:

# Install Jest
npm install --save-dev jest

# Add to package.json → "scripts"
"test": "jest"

# Run tests
npm test
Part 2

Testing arrays & objects in Jest

Four real-world scenarios — each with the function being tested and the tests that cover it.

Matchers Key matchers you'll need
toEqual()
Deep equality — compares every key/value inside an object or array. Use instead of toBe for objects.
toContain()
Checks that an array includes a specific primitive value.
toHaveLength()
Checks the .length property of arrays and strings.
toMatchObject()
Checks that an object contains a subset of expected keys — ignores extra keys.
toHaveProperty()
Checks that an object has a specific key, optionally with a specific value.
expect.arrayContaining()
Checks that an array contains all the listed items (order doesn't matter).
Example 1 Array filtering

A function that filters a list of products by category.

// shop.js 
function filterByCategory(products, category) {
  if (!Array.isArray(products)) throw new Error('products must be an array');
  return products.filter(p => p.category === category);
}

module.exports = { filterByCategory };
// shop.test.js 
const { filterByCategory } = require('./shop');

const products = [
  { id: 1, name: 'Laptop',   category: 'electronics', price: 999 },
  { id: 2, name: 'T-shirt',  category: 'clothing',     price: 29  },
  { id: 3, name: 'Phone',    category: 'electronics', price: 699 },
  { id: 4, name: 'Notebook', category: 'stationery',  price: 5   },
];

describe('filterByCategory', () => {

  test('returns only electronics', () => {
    const result = filterByCategory(products, 'electronics');
    expect(result).toHaveLength(2);
    expect(result[0].name).toBe('Laptop');
    expect(result[1].name).toBe('Phone');
  });

  test('returns empty array when no match', () => {
    const result = filterByCategory(products, 'furniture');
    expect(result).toEqual([]); // toEqual for arrays, not toBe
  });

  test('each result has the correct category', () => {
    const result = filterByCategory(products, 'electronics');
    result.forEach(item => {
      expect(item.category).toBe('electronics');
    });
  });

  test('throws when products is not an array', () => {
    expect(() => filterByCategory('oops', 'electronics'))
      .toThrow('products must be an array');
  });

});
Example 2 Object transformation

A function that formats a raw user object from an API into a cleaner shape.

// user.js 
function formatUser(raw) {
  return {
    id:       raw.user_id,
    fullName: `${raw.first_name} ${raw.last_name}`,
    email:    raw.email_address.toLowerCase(),
    isAdmin:  raw.role === 'admin',
  };
}

module.exports = { formatUser };
// user.test.js 
const { formatUser } = require('./user');

const rawUser = {
  user_id:       42,
  first_name:    'Sara',
  last_name:     'Ahmed',
  email_address: 'Sara.Ahmed@Example.COM',
  role:          'admin',
};

describe('formatUser', () => {

  test('returns a correctly shaped object', () => {
    const result = formatUser(rawUser);
    expect(result).toEqual({
      id:       42,
      fullName: 'Sara Ahmed',
      email:    'sara.ahmed@example.com',
      isAdmin:  true,
    });
  });

  test('has the expected properties', () => {
    const result = formatUser(rawUser);
    expect(result).toHaveProperty('id');
    expect(result).toHaveProperty('fullName', 'Sara Ahmed');
    expect(result).toHaveProperty('isAdmin', true);
  });

  test('lowercases the email', () => {
    const result = formatUser(rawUser);
    expect(result.email).toBe(result.email.toLowerCase());
  });

  test('isAdmin is false for non-admin role', () => {
    const result = formatUser({ ...rawUser, role: 'viewer' });
    expect(result.isAdmin).toBe(false);
  });

  test('does not expose raw field names', () => {
    const result = formatUser(rawUser);
    expect(result).not.toHaveProperty('user_id');
    expect(result).not.toHaveProperty('email_address');
  });

});
Example 3 Array of objects — sorting & mapping

A function that takes a list of orders and returns them sorted by amount, with a total added.

// orders.js 
function summariseOrders(orders) {
  const sorted = [...orders].sort((a, b) => b.amount - a.amount);
  const total  = orders.reduce((sum, o) => sum + o.amount, 0);
  return { sorted: sorted, total: total };
}

module.exports = { summariseOrders };
// orders.test.js 
const { summariseOrders } = require('./orders');

const orders = [
  { id: 'A', amount: 50  },
  { id: 'B', amount: 200 },
  { id: 'C', amount: 30  },
];

describe('summariseOrders', () => {

  test('calculates the correct total', () => {
    const { total } = summariseOrders(orders);
    expect(total).toBe(280);
  });

  test('sorts orders from highest to lowest amount', () => {
    const { sorted } = summariseOrders(orders);
    expect(sorted[0].id).toBe('B');
    expect(sorted[1].id).toBe('A');
    expect(sorted[2].id).toBe('C');
  });

  test('does not mutate the original array', () => {
    summariseOrders(orders);
    expect(orders[0].id).toBe('A'); // still in original order
  });

  test('returns zero total for empty array', () => {
    const { total, sorted } = summariseOrders([]);
    expect(total).toBe(0);
    expect(sorted).toEqual([]);
  });

  test('sorted result contains all original orders', () => {
    const { sorted } = summariseOrders(orders);
    expect(sorted).toEqual(
      expect.arrayContaining([
        expect.objectContaining({ id: 'A' }),
        expect.objectContaining({ id: 'B' }),
        expect.objectContaining({ id: 'C' }),
      ])
    );
  });

});
Example 4 Nested objects — partial matching

A function that merges a default config with user preferences. Uses toMatchObject to check partial structure.

// config.js 
const DEFAULTS = {
  theme:    'light',
  language: 'en',
  notifications: { email: true, sms: false },
};

function buildConfig(userPrefs = {}) {
  return {
    ...DEFAULTS,
    ...userPrefs,
    notifications: {
      ...DEFAULTS.notifications,
      ...(userPrefs.notifications || {}),
    },
  };
}

module.exports = { buildConfig };
// config.test.js 
const { buildConfig } = require('./config');

describe('buildConfig', () => {

  test('returns default config when called with no args', () => {
    const config = buildConfig();
    expect(config).toEqual({
      theme:    'light',
      language: 'en',
      notifications: { email: true, sms: false },
    });
  });

  test('overrides theme without touching other defaults', () => {
    const config = buildConfig({ theme: 'dark' });
    // toMatchObject only checks the keys you specify
    expect(config).toMatchObject({
      theme:    'dark',
      language: 'en',
    });
  });

  test('merges nested notifications correctly', () => {
    const config = buildConfig({ notifications: { sms: true } });
    expect(config.notifications).toEqual({
      email: true,  // kept from default
      sms:   true,  // overridden
    });
  });

  test('config has the required top-level keys', () => {
    const config = buildConfig();
    expect(config).toHaveProperty('theme');
    expect(config).toHaveProperty('notifications.email'); // dot notation!
  });

});
